Lesson 5: Enemy Movement & Collision Detection

For Students with Autism Spectrum Disorder

Modified Enemy Class with Heavy Scaffolding:

# STEP 1: Make the enemy box class Enemy(pygame.sprite.Sprite): def __init__(self, x_position, y_position): super().__init__() # The enemy is a red square self.image = pygame.Surface((30, 30)) self.image.fill((255, 0, 0)) # RED color # Where is the enemy? self.rect = self.image.get_rect() self.rect.x = x_position # Left-right position self.rect.y = y_position # Up-down position # How fast does it move? self.speed = 1 # CHANGE THIS NUMBER to make it faster

Visual Support:

Use colour-coded comments and provide a physical grid worksheet to track enemy positions.

For Students with Reading Comprehension Difficulties

Audio Support Resources:

  • Use Natural Reader to read code aloud.
  • Provide pre-recorded explanations of each code block.
  • Use visual flowcharts instead of written instructions.

Simplified Visual Collision Checker:

# LOOK at the screen - do they TOUCH? if player_box.touches(enemy_box): print("BOOM! They hit!") game_over = True

For Students with ADHD

Gamified Progress Tracker:

  • Break tasks into 5-minute "sprints".
  • Use a timer visible on screen.
  • Provide immediate visual/audio feedback for completed tasks.
  • Allow movement breaks between sprints.

Quick Win Tasks:

  • Make enemy bigger (change one number)
  • Make enemy faster (change one number)
  • Change enemy colour (change three numbers)
  • Add a sound effect (one line of code)

For Students with Intellectual Developmental Disorders

Pre-Written Template with Fill-in-the-Blanks:

# Enemy Setup - Just change the NUMBERS enemy_color = (255, 0, 0) # Change these 3 numbers for colour enemy_size = 30 # Change this for size enemy_speed = 2 # Change this for speed class Enemy(pygame.sprite.Sprite): # Teacher has written this part # You just need to RUN it and CHANGE the numbers above

Physical "Unplugged" Activities:

  • Use toy cars on a grid to demonstrate collision.
  • Have student be the "computer" and move objects when you give commands.
  • Use large printed code cards to sequence the logic.
